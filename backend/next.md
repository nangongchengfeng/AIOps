---
  Agent 模式需要增加的能力

  1. 自主观测能力（Observation）

  不只是等 webhook，还要主动发现问题：
  - 定时轮询 Prometheus 查异常指标
  - 主动检查告警静默期是否即将结束
  - 监控多个数据源（Prometheus + Loki 日志等）

  2. 动作执行能力（Action）

  不只是分析，还能自动执行操作：
  - 调用 K8s API 扩容/重启 Pod
  - 触发告警静默（AlertManager API）
  - 执行预定义的 Runbook 脚本
  - 发送通知到钉钉/飞书/Slack

  3. 反思与记忆（Memory/Reflection）

  - 历史分析记忆：同样问题下次更快
  - 反馈循环：用户修正后，调整后续分析
  - 工具使用记忆：记住哪些动作有效

  4. 规划能力（Planning）

  多步骤推理：
  - "磁盘满了" → 先查大文件 → 再清理日志 → 最后通知
  - 而不是只给出建议
